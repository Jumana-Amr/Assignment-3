Petya Ransomeware

How it spreads:

Once the victim is infected, the Petya exploits the vulnerability CVE-2017-0144 in Microsoft’s implementation of the Server Message Block protocol. This makes the operating system incapable of locating files. Petya installs its own boot loader, overwriting the system's master boot, then it encrypts some of the data on the computer, and sends the victim a message to the conduct system reboot. The ransomware demands a Bitcoin payment in order to decrypt the hard drive.

How to respond:

Once a user is infected, we need to start containment stage, the infected computer should be turned off and disconnect from the network. Petya waits about an hour after infecting a system before rebooting and displaying a message that the file system is being “repaired.” If the machine is turned off immediately, some files may be saved, and disconnecting it from the network will prevent its spread.
